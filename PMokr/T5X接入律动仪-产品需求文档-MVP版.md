# T5X 接入律动仪 - MVP 版本产品需求文档

> 本文档基于 **MVP 演示 Demo**（`律动仪接入-MVP演示.html`）的实际功能进行整理，作为研发实现与验收依据。

> **文档版本**：MVP 版本（手机端）

## 一、需求背景

### 1.1 业务背景

- **主设备定位**：评估设备，核心能力包括：
  - **体成分**：体重、体脂率、肌肉量等
  - **体态**：高低肩、头侧歪、腿型、骨盆等
  - **体适能**：平衡、柔韧性、握力、肺活量等
  - **FMS 评估**：七项动作筛查及总分
- **评估后产出**：根据用户评估数据生成**训练方案**。
- **律动仪接入目标**：评估完成后，输出**使用律动仪完成的训练课程**；用户可在**设备端**查看课程，课程支持**下发到律动仪**进行训练。

### 1.2 目标用户与场景

- **主要用户**：在门店使用评估设备的终端用户（如健身房会员）。
- **使用终端**：评估设备端系统（手机端/移动端），以及律动仪硬件。
- **核心场景**：完成体成分/体态/体适能/FMS 评估 → 查看推荐课程 → 选择律动仪课程 → 在律动仪上执行训练。

### 1.3 系统角色

| 角色 | 职责 |
|------|------|
| **后台管理系统** | 课程创建、配置、上架；同步/下发课程列表；提供课程详情、课程配置、推荐课程接口 |
| **设备端系统** | 评估能力入口、展示推荐课程、请求课程配置/详情、向律动仪下发训练指令、展示训练状态 |
| **律动仪设备** | 接收训练指令，执行并上报执行状态 |

### 3.5 训练页

#### 3.5.1 页面结构

- **课程名称**：16px，居中
- **进度条**（仅完整课程显示）：
  - 高度 6px，蓝紫渐变填充
  - 宽度随当前动作进度变化（1/3 → 2/3 → 3/3）
- **进度文案**（仅完整课程显示）：
  - 格式：「动作 1/3 · 深蹲律动」
  - 随当前播放动作更新
- **视频播放区**：
  - **竖屏视频样式**：9:16 比例，全宽，居中
  - 视频元素：`object-fit: contain`，支持播放控制（`controls`）
  - 背景：黑色
- **训练状态**：
  - 进行中：蓝底 + 旋转图标 +「训练进行中…」
  - 已结束：绿底 +「训练已结束」
- **说明文案**：
  - 进行中：「课程已下发至律动仪，请跟随视频完成训练。」
  - 提前结束：「您已提前结束本次训练。」
  - 正常结束：「本次训练已结束，律动仪已停止。」
- **操作按钮**：
  - 「立即结束」：训练进行中时显示
  - 「返回课程列表」：训练结束后显示

#### 3.5.2 完整课程训练流程

1. 用户点击「开始训练（完整课程）」
2. 进入训练页，显示课程名称
3. 显示进度条与进度文案（初始：动作 0/3）
4. 依次播放课程中所有动作视频：
   - 播放第 1 个视频 → 进度文案更新为「动作 1/3 · 深蹲律动」，进度条填充至 33.3%
   - 播放第 2 个视频 → 进度文案更新为「动作 2/3 · 臀桥律动」，进度条填充至 66.7%
   - 播放第 3 个视频 → 进度文案更新为「动作 3/3 · 侧向律动」，进度条填充至 100%
5. 所有视频播放完成后：
   - 状态改为「训练已结束」
   - 进度文案更新为「动作 3/3 已完成」
   - 隐藏「立即结束」，显示「返回课程列表」

#### 3.5.3 单动作训练流程

1. 用户点击动作项的「开始训练」→ 进入单动作详情页
2. 点击「开始训练」→ 进入训练页
3. 不显示进度条与进度文案
4. 播放该动作的视频
5. 视频播放完成后：
   - 状态改为「训练已结束」
   - 隐藏「立即结束」，显示「返回课程列表」

#### 3.5.4 立即结束功能

- **显示时机**：训练进行中时显示「立即结束」按钮
- **交互**：点击后：
  - 暂停并清空当前视频
  - 状态改为「训练已结束」
  - 说明文案改为「您已提前结束本次训练。」
  - 完整课程时，进度文案改为「已提前结束」
  - 隐藏「立即结束」，显示「返回课程列表」

## 六、技术规范

### 6.1 视频播放

- **格式**：MP4（H.264）
- **比例**：竖屏 9:16（适配竖版视频内容）
- **播放控制**：使用 HTML5 `<video>` 原生控件（`controls`）
- **预加载**：动作缩略图使用 `preload="metadata"`，静音加载首帧
- **路径**：视频文件与 HTML 同目录或子目录，使用相对路径（如 `./001-站姿.mp4`）

### 6.2 页面切换

- **方式**：单页应用（SPA），使用 JavaScript 控制页面显示/隐藏
- **页面标识**：`page-entry`、`page-list`、`page-detail`、`page-single-action`、`page-training`
- **状态管理**：使用全局变量存储当前课程/动作信息

### 6.3 空闲倒计时

- **实现**：`setInterval` 每秒更新倒计时与进度条
- **重置触发**：`mousedown`、`touchstart` 事件
- **倒计时结束**：调用 `showPage('entry')` 返回入口页

## 七、异常状态与边界条件处理

### 7.1 网络异常处理

#### 7.1.1 接口请求失败

| 场景 | 处理方式 | 用户提示 |
|------|----------|----------|
| **课程列表接口失败** | 显示空状态页面，提供「重试」按钮 | 「课程加载失败，请检查网络后重试」 |
| **课程详情接口失败** | 停留在课程列表页，显示 Toast 提示 | 「课程详情加载失败，请稍后重试」 |
| **推荐课程接口失败** | 仅显示「全部课程」区块，隐藏「推荐课程」区块 | 无提示（静默降级） |
| **请求超时（>10秒）** | 自动重试 1 次，仍失败则显示错误提示 | 「请求超时，请检查网络连接」 |

**实现要求**：
- 所有接口请求需设置超时时间（建议 10 秒）
- 失败时显示明确的错误提示，提供重试机制
- 网络错误与业务错误（如 404、500）需区分提示

#### 7.1.2 无网络连接

- **检测**：使用 `navigator.onLine` 或监听 `online/offline` 事件
- **提示**：页面顶部显示横幅提示「网络连接已断开，请检查网络设置」
- **行为**：已加载的数据可正常浏览，新请求提示网络错误

---

### 7.2 视频播放异常

#### 7.2.1 视频加载失败

| 场景 | 处理方式 | 用户提示 |
|------|----------|----------|
| **视频文件不存在（404）** | 跳过该视频，播放下一段；若无下一段则结束训练 | 「视频加载失败，已跳过该动作」 |
| **视频格式不支持** | 显示错误提示，提供「跳过」按钮 | 「视频格式不支持，请检查视频文件」 |
| **视频文件损坏** | 尝试播放，失败后跳过 | 「视频播放失败，已跳过该动作」 |
| **视频加载超时（>30秒）** | 自动跳过，播放下一段 | 「视频加载超时，已跳过该动作」 |

**实现要求**：
- 监听视频 `onerror` 事件，捕获加载失败
- 完整课程训练时，单个视频失败不应中断整个训练流程
- 单动作训练时，视频失败应显示错误并允许返回

#### 7.2.2 视频播放异常

| 场景 | 处理方式 | 用户提示 |
|------|----------|----------|
| **播放中断（用户切换应用）** | 暂停视频，恢复时继续播放 | 无提示（系统行为） |
| **内存不足** | 降低视频质量或提示用户关闭其他应用 | 「设备内存不足，请关闭其他应用后重试」 |
| **视频时长异常** | 正常播放，但需记录异常日志 | 无提示（后台记录） |

**实现要求**：
- 监听 `visibilitychange` 事件，页面隐藏时暂停视频
- 视频播放前检查可用内存（如可能）
- 记录异常视频的 URL 和错误信息，便于排查

#### 7.9.1 训练中断处理

| **应用切换到后台** | 暂停视频播放，恢复时继续播放 | 无提示（系统行为） |
| **设备锁屏** | 暂停视频播放，解锁后继续播放 | 无提示（系统行为） |
| **来电中断** | 暂停视频播放，通话结束后提示是否继续 | 「训练已暂停，是否继续？」 |

- **场景**：用户意外离开训练页后返回
- **处理**：MVP 版本不保存训练进度，返回后重新开始
- **提示**：无（后期版本可支持进度保存）

### 7.10 异常处理通用原则

1. **用户友好**：所有错误提示使用通俗易懂的语言，避免技术术语
2. **可恢复**：关键操作失败时提供「重试」或「返回」选项
3. **降级处理**：非核心功能失败时不影响核心流程（如推荐课程失败时仍可浏览全部课程）
4. **日志记录**：所有异常需记录日志（错误类型、时间、用户信息），便于排查
5. **性能优先**：异常处理不应影响页面性能，避免阻塞主线程
