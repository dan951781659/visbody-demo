# AI 报告解读模块 - 产品需求文档

> **编制说明**：本文档面向体测报告 AI 解读功能，从健康问题、形体美学、运动能力三大维度输出综合结论，支持多 SKU、多指标组合的全场景覆盖。  
> **关联文档**：《综合结论规则库》《AI报告解读-综合结论规则库-完整版.csv》

---

## 一、需求背景

### 1.1 业务背景

- 公司生产**静态体测仪**、**动态体适能 FMS 评估设备**，多款 SKU 覆盖不同客户场景
- 不同 SKU 可评估的项目不同，用户可能只测 1 项，也可能测多项组合
- 测量报告仅展示单一指标数据，用户无法从**健康问题**、**形体美学**、**运动能力**三大维度理解自身关键问题与正常表现

### 1.2 目标用户与场景

| 维度 | 说明 |
|------|------|
| **目标用户** | 使用体测设备的终端用户（C 端），以及瑜伽/普拉提、减肥机构、运动康复机构等 B 端客户 |
| **使用场景** | 用户在设备完成测量后，在报告详情页查看「AI 深度解读」，按三大维度理解自身问题 |
| **客户差异** | 瑜伽/减肥机构更关注形体美学与健康问题；运动康复/能力提升机构更关注运动能力 |

### 1.3 设计目标

1. **全场景覆盖**：无论用户测 1 项还是多项，系统均能输出有效解读
2. **综合结论优先**：从多指标推导综合征/体态类型（如中心性肥胖、上交叉综合征），而非单指标罗列
3. **三大维度输出**：健康问题、形体美学、运动能力
4. **高扩展性**：新增 SKU 或评估指标时，仅需更新配置表，无需修改核心代码

### 1.4 当前痛点

- 基于决策树的解读逻辑无法覆盖所有用户测量场景
- 存在无法输出结论或结论与用户实际情况不符的情况
- 输出多为单指标解释，缺乏多指标综合推导

## 二、功能概述

### 2.1 核心能力

| 能力 | 说明 |
|------|------|
| **数据识别** | 自动识别报告来源 SKU、本次测量项目，建立白名单 |
| **异常判定** | 按各指标标准范围判定异常状态及严重度 |
| **综合结论匹配** | 根据指标组合匹配综合征/体态类型（中心性肥胖、上交叉综合征等） |
| **维度输出** | 按三大维度组织综合结论与单指标兜底，输出结构化解读 |
| **LLM 润色** | 将结构化结论整合为自然、连贯的解读文案 |

### 2.2 功能入口

| 项目 | 说明 |
|------|------|
| **载体** | H5/App 报告详情页、PDF 报告内嵌 |
| **入口** | 「AI 深度解读」卡片或 Tab，点击展开三大维度内容 |
| **前置条件** | 报告至少包含 1 个有效测量指标 |

- **SKU 扩展**：在 `SKU_Indicator_Compatibility` 配置表中新增 SKU 与指标的兼容关系
- **指标扩展**：在 `Indicator_Base` 表中新增指标定义，在《综合结论规则库》中补充其与综合结论的映射
- 系统运行时动态读取配置，无需改代码即可支持新 SKU 与新指标

## 五、综合结论规则（概要）

> 详细规则定义见《综合结论规则库》。

### 5.1 设计原则

- **综合结论优先**：多指标组合匹配综合征/体态类型，输出综合结论
- **单指标兜底**：未命中任何规则的异常指标，按单指标解读兜底
- **证据可溯**：每个综合结论附带参与指标及实测值

### 5.2 典型综合结论示例（异常类）

| 综合结论 | 必要指标组合 | 所属维度 |
|----------|--------------|----------|
| 中心性肥胖 | BMI 正常 且 腰臀比超标 | 健康问题、形体美学 |
| 上交叉综合征 | 头前引异常 且 (左圆肩 或 右圆肩) 异常 | 健康问题、形体美学、运动能力 |
| 下交叉综合征 | 骨盆前倾 且 (腰椎过伸 或 髋屈肌紧张) | 健康问题、形体美学、运动能力 |
| 肌肉型肥胖 | 体脂率偏高 且 肌肉量充足/超标 | 健康问题、形体美学、运动能力 |
| 隐性肥胖 | BMI 正常 且 体脂率偏高 且 肌肉量偏低 | 健康问题、形体美学 |
| 跌倒风险偏高 | 闭眼平衡差 且 (年龄>60 或 病史) | 健康问题、运动能力 |

### 5.3 正常结论（激励类）

当某维度下**已测指标均为正常**时，输出激励性正常结论，而非留空。

| 正常结论 | 触发条件 | 所属维度 |
|----------|----------|----------|
| 体成分表现良好 | 体成分模块已测且均正常 | 健康问题、形体美学 |
| 体态表现良好 | 体态模块已测且均正常 | 健康问题、形体美学、运动能力 |
| 体围比例良好 | 体围模块已测且均正常 | 形体美学 |
| 关节活动度良好 | 肩/髋/颈功能已测且均正常 | 运动能力、健康问题 |
| 脊柱状态良好 | 脊柱模块已测且均正常 | 健康问题、运动能力 |
| FMS 动作模式良好 | FMS 已测项目均正常 | 运动能力 |
| 体适能表现良好 | 体适能已测项目均正常 | 运动能力 |
| 平衡能力良好 | 平衡模块已测且均正常 | 运动能力、健康问题 |
| 臀型理想 | BT_01 为蜜桃臀或圆形臀 | 形体美学 |
| 整体表现优秀 | 所有已测模块均无异常 | 健康问题、形体美学、运动能力 |

### 5.4 与单指标的关系及匹配顺序

- **异常匹配**：若指标 A、B 共同命中「中心性肥胖」，则**不再**单独输出「腰臀比超标」「BMI 正常」的单指标解读
- **单指标兜底**：若某指标未参与任何综合结论，则进入单指标兜底池
- **正常结论**：异常规则匹配完成后，若某维度无任何异常结论，则输出对应 NORM_* 正常结论
- 优先级与去重逻辑见《综合结论规则库》

## 八、边界条件与规则

### 8.1 数据层约束

| 规则 | 说明 |
|------|------|
| 白名单 | 仅解读本次报告中有数值的指标 |
| 禁止推断 | 不得根据已有指标推测未测量指标状态 |
| 异常前置 | LLM 仅接收已判定异常/正常的条目 |

### 8.2 专业边界

| 规则 | 说明 |
|------|------|
| 不下诊断 | 使用「可能存在…风险」「建议关注」等表述 |
| 建议通用化 | 建议为普适性运动/生活方式建议，不涉及处方 |

### 8.3 异常场景与无法输出结论

**8.3.1 场景清单与处理策略**

| 序号 | 场景描述 | 触发条件 | 系统处理 | 用户侧展示 |
|------|----------|----------|----------|------------|
| 1 | 报告无任何测量数据 | report_data 为空或所有指标值为 null/空 | 不调用规则引擎与 LLM | 「暂无有效测量数据，请完成测量后再查看 AI 解读。」 |
| 2 | 报告仅有表头/元数据，无有效数值 | 所有指标 value 为 --、N/A、空字符串 | 同上 | 同上 |
| 3 | 测量失败导致数据缺失 | 衣着/头发/姿势不标准导致合成失败 | 仅对有效指标解读；失败模块不输出 | 「部分项目因测量条件限制未获取有效数据，当前解读仅基于已完成测量的项目。建议按要求重新测量以获得完整解读。」 |
| 4 | 仅测 1 项且该项无法判定 | 设备返回值为「待定」「无效」 | 无法给出结论 | 「当前测量数据不足以生成解读，建议重新测量或完成更多测量项目。」 |
| 5 | 测量的指标不在规则库中 | 新 SKU/新指标尚未配置规则 | 无法匹配任何规则 | 「您本次测量的项目暂不支持 AI 解读，我们将持续完善。您可先查看原始报告数据。」 |
| 6 | 测量指标组合无规则且单指标均正常 | 如仅测 BC_07 蛋白质且正常 | 输出正常结论 | 输出 NORM_BC_001 或对应模块正常结论 |
| 7 | 测量指标组合无规则且单指标均异常 | 冷门指标异常但无兜底规则 | 无法给出结论 | 「当前测量项目暂无对应解读规则，建议联系客服或等待后续版本更新。」 |
| 8 | 数据格式异常 | 接口格式错误、编码异常、字段缺失 | 不调用 LLM，记录日志 | 「解读生成失败，请稍后重试。若重复出现请联系客服。」 |
| 9 | SKU 无法识别 | 设备型号不在支持的 SKU 列表 | 尝试通用指标匹配；失败则提示 | 「当前设备型号暂不支持 AI 解读」 |
| 10 | 所有已测指标均为正常 | 规则匹配无异常结论 | 输出正常结论 | 输出对应维度的 NORM_* 正常结论 |
| 11 | 年龄/性别等必要参数缺失 | 部分规则需性别、年龄判定 | 降级为不依赖该参数的规则 | 尽量输出，不足处说明 |
| 12 | LLM 调用超时或失败 | 网络、服务不可用 | 降级：仅输出结构化结论列表 | 「当前为简化版解读，完整版生成中，请稍后刷新。」 |

**8.3.2 处理流程示意**

```
报告数据输入 → [1]数据有效性校验（无数据→固定提示；部分失败→过滤无效项）
    → [2]SKU与指标白名单校验（不可识别→尝试通用匹配）
    → [3]规则匹配（无命中→对应提示；有异常→输出异常；无异常→输出正常结论）
    → [4]LLM润色（成功→完整解读；失败→降级输出）
```

**8.3.3 兜底话术库（供前端/配置）**

| 场景类型 | 话术 |
|----------|------|
| 无数据 | 暂无有效测量数据，请完成测量后再查看 AI 解读。 |
| 部分失败 | 部分项目因测量条件限制未获取有效数据，当前解读仅基于已完成测量的项目。建议按要求重新测量以获得完整解读。 |
| 项目不支持 | 您本次测量的项目暂不支持 AI 解读，我们将持续完善。您可先查看原始报告数据。 |
| 规则未覆盖 | 当前测量项目暂无对应解读规则，建议联系客服或等待后续版本更新。 |
| 系统异常 | 解读生成失败，请稍后重试。若重复出现请联系客服。 |
| 设备不支持 | 当前设备型号暂不支持 AI 解读。 |

### 8.4 医疗用词规范（无医疗资质）

公司从事健康服务，无医疗资质，结论中需避免医疗诊断、治疗类表述。

**禁用词及替换建议**

| 禁用词/表述 | 替换建议 |
|-------------|----------|
| 诊断、确诊、罹患、患病 | 删除或改为「建议关注」「存在…倾向/风险」 |
| 疾病、病症 | 改为「风险」「负荷」；「综合征」在康复领域可保留 |
| 治疗、治愈、康复（医疗语境） | 改为「改善」「调整」「训练」「手法放松」 |
| 处方、用药、医嘱 | 删除 |
| 症状 | 改为「表现」「身体感受」 |
| 临床、就诊、就医、咨询医生 | 改为「专业评估」「专业筛查」「建议咨询专业人士」 |
| 医生、医院 | 改为「专业人士」「专业机构」 |
| 物理治疗 | 改为「专业体态矫正指导」「专业评估」 |
| 检查、化验 | 改为「评估」「检测」「筛查」 |

**表述原则**

1. **建议类**：多用「建议…」「可考虑…」「有助于…」，少用「应」「必须」
2. **风险类**：用「存在…倾向」「需关注…」「可能影响…」
3. **专业介入**：用「专业评估」「专业筛查」「专业人士指导」，避免「就诊」「看病」「治疗」
4. **报告免责**：结论末尾可统一加「以上内容仅供参考，不构成医疗建议。」

## 九、验收标准

1. 支持《体成分参数对比-兼容性表格》中全部 16 款 SKU 及所列评估项目
