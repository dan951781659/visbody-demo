# 报告列表批量导出（Excel/CSV）- 产品需求文档

> **编制说明**：本文档面向教育/科研客户，支持将报告列表数据批量导出为 Excel 或 CSV 格式，便于复制数据做研究用途。  
> **关联原型**：`报告列表批量导出-研发演示Demo.html`（开发前演示用）。

---

## 一、需求背景

### 1.1 业务背景

- 客户使用体态/体测设备生成测量报告，报告以 PDF 形式呈现，包含体态评估、身体围度、肩部功能等多类数据
- 教育机构、科研单位等客户需要将多份报告的数据汇总为结构化表格，用于课题研究、论文撰写、教学案例等
- 当前报告列表仅支持单条「下载 PDF」或「查看详情」，无法一次性导出多条报告的结构化数据，客户需逐条打开 PDF 手工摘录，效率低且易出错

### 1.2 目标用户与场景

| 维度 | 说明 |
|------|------|
| **目标用户** | 教育/科研客户的管理员或研究人员（使用管理后台） |
| **使用场景** | 在报告列表页筛选条件后，勾选多条报告，一键导出为 Excel 或 CSV，在本地进行统计分析 |
| **核心价值** | 降低数据整理成本，提高研究效率；导出字段与 PDF 报告内容一一对应，保证数据完整性 |

### 1.3 需求目标

1. 在**报告列表页**增加「批量导出」能力，支持勾选多条报告后导出为 **Excel（.xlsx）** 或 **CSV（.csv）**
2. 导出的数据需**覆盖 PDF 报告中的全部信息**（用户信息、体态评估、体围测量、肩部功能等），字段定义见本文档「导出数据结构」
3. 明确**边界条件**（单次导出条数上限、空值/缺失项处理等），并在交互上给予明确提示

---

## 二、功能概述与入口

### 2.1 两种导出方式（解决「单页勾选有限」与「全选影响性能」的矛盾）

| 方式 | 说明 | 适用场景 | 性能与限制 |
|------|------|----------|------------|
| **方式一：按勾选导出** | 用户勾选列表中的行，仅导出**已勾选**的报告 | 少量、精确指定（如十几条、几十条） | 单次上限 500 条；不依赖「全选当前筛选结果」，避免前端大量勾选导致卡顿 |
| **方式二：按条件导出** | 用户设置**筛选条件**（时间段、性别、测量项目等），点击「导出当前筛选结果」，后端按条件查询并生成文件 | 一次性导出某时间段、某性别或某测量项目的全部数据 | **无需在前端勾选**，不占用前端性能；建议一律走**异步任务**（提交 → 后台生成 → 完成后通知下载），单次上限可单独约定（如 500/2000，超量时提示或自动分批） |

- **设计原则**：需要导出大量数据时，用「按条件导出」；只需导出少量时，用「按勾选导出」。不提供「全选当前筛选结果」后一次性导出（避免前端勾选成千上万条导致性能问题）。
- **筛选条件**（与列表筛选一致，用于按条件导出）：
  - **时间段**：开始日期、结束日期（与现有列表筛选共用）
  - **性别**：全部 / 男 / 女
  - **测量项目**：全部 / 体态评估 / 身体围度 / 肩部功能（与列表「测量项目」一致，可多选或单选，以实际产品为准）

### 2.2 功能定位（汇总）

- **按勾选导出**：列表支持行级复选框 + 表头「全选本页」；导出格式可选 Excel 或 CSV；数据范围 = 当前已勾选行；未勾选时不可发起。
- **按条件导出**：筛选区支持时间段、性别、测量项目；导出格式可选 Excel 或 CSV；数据范围 = 当前筛选条件下的全部报告（由后端按条件查询）；无需勾选，建议异步生成。

### 2.3 功能入口与页面形态

| 项目 | 说明 |
|------|------|
| **载体** | 管理后台 - 报告列表页（与现有「数据预览」同页） |
| **筛选条件** | **开始日期 - 结束日期**（日期范围，带日历选择）、**按邮箱/姓名**（搜索框）、**搜索**按钮（放大镜图标）、**数据预览**按钮；性别、测量项目仅在「批量导出」弹窗内「按条件导出」时设置，列表页不展示 |
| **导出入口** | 筛选区「数据预览」旁：<br>• **批量导出**：按勾选导出（下拉「导出为 Excel」「导出为 CSV」），未勾选时置灰或提示；<br>• **按条件导出**：按当前筛选结果导出（下拉「导出为 Excel」「导出为 CSV」），点击后可先展示「当前筛选结果约 N 条，确认导出？」再提交异步任务 |
| **前置条件** | 按勾选：至少勾选 1 条；按条件：已设置筛选条件（时间段必填或默认），点击后确认条数再提交 |

### 3.5 数据组织方式建议（供研发实现参考）

- **方案 A - 宽表（推荐）**：一行 = 一次完整检测（含体态+体围+肩部功能），所有子项作为列。若某次检测仅有体态无体围，则体围相关列为空。
- **方案 B - 多行**：一行 = 一类报告（体态 / 体围 / 肩部功能），同一检测可占 1～3 行，通过「检测时间+用户ID」关联。

## 四、边界条件与规则

### 4.1 单次导出数量上限

| 规则项 | 建议值 | 说明 |
|--------|--------|------|
| 单次导出条数上限 | **500 条**（可配置） | 超过时提示：「单次最多导出 500 条报告，请缩小筛选范围或分页勾选后分批导出。」并禁止发起导出 |
| 未勾选任何一条 | 不允许导出 | 按钮置灰或点击后 Toast：「请至少选择 1 条报告后再导出。」 |

### 4.2 筛选与导出范围

- **按勾选导出**：导出范围以**当前已勾选的行为准**；「全选」仅针对**当前页**（与「20 条/页」等分页一致）。不提供「全选当前筛选结果」后一次性导出，避免前端勾选大量行导致性能问题。
- **按条件导出**：导出范围以**当前筛选条件**为准（时间段、性别、测量项目、搜索等），由**后端按条件查询**生成文件，不依赖前端勾选。筛选条件与列表共用，用户先筛选、再点击「按条件导出」即可。

### 4.3 性能与超时

- **按勾选导出**：条数 ≤ 单次上限（如 500）时可**同步**生成并下载；超过上限时禁止发起并提示「单次最多导出 500 条，请缩小范围或使用『按条件导出』」。
- **按条件导出**：建议**一律走异步任务**（提交导出任务 → 后台按条件查询并生成 → 完成后通知下载/站内消息），避免大结果集阻塞浏览器；单次上限可单独约定（如 500 或 2000），超量时提示「当前筛选结果超过 N 条，将分批生成」或限制最大条数。前端需展示「当前筛选结果约 N 条」供用户确认后再提交。

### 4.4 空值、缺失项与编码

- 未测量的项目在 Excel/CSV 中留空或写「-」「N/A」，统一约定一种，避免混用
- CSV 编码：**UTF-8**（含 BOM 以便 Excel 正确识别中文）
- Excel 单元格格式：数值列用数字格式，文本列用文本，日期时间统一格式（如 yyyy-mm-dd hh:mm:ss）

### 4.5 敏感信息

- 导出内容中的用户联系方式需与列表一致（脱敏），不得在导出文件中恢复明文手机号/邮箱

## 五、交互与视觉说明（与演示 Demo 一致）

### 5.1 列表变更

- 表头增加一列「勾选」：复选框，支持「全选当前页」「取消全选」
- 每一行首列增加复选框，勾选后计入「已选 N 条」
- 已选条数实时展示在「批量导出」按钮旁或下拉入口上（如「批量导出 (3)」）

### 5.2 筛选区（报告列表页）

- **日期范围**：开始日期 - 结束日期（中间以「-」连接，右侧可带日历图标点击唤起日期选择）
- **按邮箱/姓名**：搜索框占位符「按邮箱/姓名」，输入后点击搜索按钮查询
- **搜索按钮**：蓝色按钮、放大镜图标，点击后按当前日期范围 + 邮箱/姓名条件刷新列表
- **数据预览**：蓝色主按钮「数据预览」，按当前筛选条件预览数据
- **批量导出**：蓝色主按钮「批量导出」，点击后进入弹窗选择「按勾选导出」或「按条件导出」；按条件导出时在弹窗内再设置时间段、性别、测量项目
- 列表「共 N 条」与当前筛选条件联动

### 5.3 导出入口

- 位置：筛选区「数据预览」旁
- **批量导出（按勾选）**：主按钮「批量导出」或「批量导出 ▼」；下拉「导出为 Excel」「导出为 CSV」。未勾选时置灰或 Toast「请至少选择 1 条报告后再导出」；勾选 > 500 时 Toast「单次最多导出 500 条，请缩小范围或使用『按条件导出』」。
- **按条件导出**：按钮「按条件导出」或「按条件导出 ▼」；下拉「导出为 Excel」「导出为 CSV」。点击后弹窗或 Toast 展示「当前筛选结果约 N 条，确认导出？」用户确认后提交异步任务，提示「导出任务已提交，完成后将通知下载」。

### 5.4 导出过程与结果

- 点击后：按钮进入 loading（如「导出中…」），禁止重复点击
- 成功：浏览器下载文件，文件名建议包含时间戳与条数，如 `报告数据导出_20260130_15条.xlsx`
- 失败：Toast 提示具体原因（如「导出失败，请稍后重试」或服务端返回信息）

### 5.5 导出后 Excel 展示样式

导出的 Excel 文件需符合以下展示规范，便于教育/科研用户直接用于统计与排版：

| 项目 | 规范 |
|------|------|
| **表头行** | 第 1 行为列名（与本文档「三、导出数据结构」字段名一致）；建议加粗、浅灰底（如 #f5f5f5），行高略高于数据行 |
| **冻结窗格** | 冻结首行（表头），横向滚动时表头始终可见 |
| **列顺序** | 按「基础信息 → 体态评估 → 身体围度 → 肩部功能」顺序排列；同一类内按字段表顺序 |
| **数值列** | 角度、长度、围度等为数字格式，保留 1 位小数（如 3.8、177.9）；整数列（如年龄、体态得分）无小数 |
| **日期时间列** | 格式统一为 `yyyy-mm-dd hh:mm:ss`（如 2026-01-22 11:06:14），单元格格式为「日期」或「自定义」 |
| **文本列** | 用户ID、姓名、评估结论、风险提示等为文本格式，避免被 Excel 自动转为数字或日期 |
| **空值** | 未测量或缺失项统一显示为「-」或留空（与 4.4 约定一致） |
| **列宽** | 表头与内容可读；建议数值列 10–12 字符宽，长文本列（如风险提示）可 25–35 字符宽或自动适应 |
| **多 Sheet（可选）** | 若单表列数过多，可按「基础信息 + 体态」「基础信息 + 体围」「基础信息 + 肩部功能」拆为多个 Sheet，每 Sheet 首列保留用户ID/检测时间等关键信息便于关联 |

**示意（首行与 2 行数据）：**

| 用户ID | 用户姓名 | 性别 | 身高_cm | 年龄_岁 | 检测时间 | 体态得分 | 头前引_cm | 头前引_评估结论 | … |
|--------|----------|------|---------|---------|----------|----------|-----------|----------------|---|
| koko-sss | - | 男 | 168 | 25 | 2026-01-22 11:06:14 | 78 | 3.8 | 存在明显头前引倾向 | … |
| 刘 | - | 男 | 170 | 28 | 2026-01-22 14:33:02 | 82 | 2.1 | 存在头前引倾向 | … |

CSV 导出时列顺序、列名与空值规则与 Excel 一致；无表头样式与冻结，仅保证编码 UTF-8（含 BOM）及分隔符（逗号）。

## 六、验收标准（摘要）

1. **按勾选导出**：报告列表支持行级勾选 + 表头全选当前页；「批量导出」仅对已勾选条数生效；未勾选时提示「请至少选择 1 条报告」；勾选 > 500 时提示并禁止。
2. **按条件导出**：筛选区支持时间段、性别、测量项目；「按条件导出」以当前筛选结果为范围，无需勾选；提交前展示「当前筛选结果约 N 条」并确认；按条件导出走异步任务，完成后可下载或站内通知。
3. 两种方式均支持「导出为 Excel」或「导出为 CSV」，文件可正常打开、中文无乱码。
4. 导出文件包含本文档「三、导出数据结构」中约定的全部字段；与 PDF 报告内容一致（含基础信息、体态、体围、肩部功能）。
5. 单次导出条数上限（按勾选 500、按条件可单独约定）、异步/同步策略、超时与异常提示符合「四、边界条件」约定。
6. 用户联系方式在导出文件中与列表一致（脱敏），无明文敏感信息。
