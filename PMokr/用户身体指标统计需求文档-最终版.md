## 三、数据规则
### 3.1 数据隔离
移动端-H5

+ 按**客户**隔离：仅统计用户在当前客户下的测量记录及对应单项参数；
+ 不同客户之间数据互不可见

管理后台PC

+ 按照客户-门店 进行数据隔离，统计 用户在当前门店下所有测量记录（总门店可查看子门店所有数据，子门店只能查看子门店） 对应 产生的单项参数记录；  
	

### 3.2 测量数据取值规则
+ 若当天做了多次测量，则只取一次数据；
+ 若当天多次测量的设备类型一样，则按时间：取最近一次的测量值；
+ 若当天多次测量的设备类型不一样，则先按照第2条规则取同一类型设备最近一次的测量值，再按照设备优先级，取优先级高的设备的数据；
    - 设备优先级以客户门店信息管理中配置的优先级为准；<!-- 这是一张图片，ocr 内容为： -->

## 四、报告类型与指标卡片
### 4.1 报告类型
+ 显示用户测量过的所有参数的数据，若用户没有测量过该类型的数据，则不显示该table。如：没有测量过体适能则不显示体适能的table；
+ 切换报告类型的时候，显示该类型下的参数统计；
+ 体成分、体态、体围、腰腹围度、肩部、颈部、平衡、FMS、体适能等展示样式均如图的体成分所示；

### 4.3 指标规则
+ **默认选中**：当前报告类型下前 7 个指标（不足则全部选中）
+ **选择方式**：点击卡片选中 / 取消选中
+ **联动**：选中变化后，下方「数据统计」区的图例与折线图同步更新
+ **数据来源展示**：仅当所选参数的数据来源设备**不一致**时，在卡片上显示「?」图标；点击后弹框展示该条数据的数值及来源设备型号，交互与图表「查看详细数据与设备来源」一致。设备一致时不展示「?」

## 五、数据统计与图表
### 5.1 时间范围
+ **默认**：最近 1 个月
+ **组件**：开始日期、结束日期（日期选择器）
+ **格式**：展示与接口使用 **YYYY/MM/DD **（如 2025/05/26）
+ 最长：1年

### 5.2 多指标与多图表
+ **同单位一图**：单位相同的指标在同一折线图中对比；单位不同则分多图
+ **图例**：与选中指标一致，每指标一色（如体重蓝、体脂率绿、去脂体重黄等）
+ **图表标题**：按单位分组时，每组可带标题，如「kg」· 同类型合并展示；采用双 Y 轴时标题可带「（左/右轴）」提示

### 5.3 折线图交互（与原型一致）
+ **悬停模式**：`intersect: false`、`mode: 'index'`，沿时间轴移动即可触发，无需精确对准数据点
+ **Tooltip（当天所有选中指标）**：
    - 触发：鼠标悬停某天的数据点（任意一条线均可）
    - <font style="background-color:#E8F7CF;">内容：</font>**<font style="background-color:#E8F7CF;">当天日期时间</font>**<font style="background-color:#E8F7CF;">（YYYY/MM/DD HH:mm:ss）+ </font>**<font style="background-color:#E8F7CF;">当前选中的全部指标及对应数值</font>**<font style="background-color:#E8F7CF;">（含单位）+ </font>**<font style="background-color:#E8F7CF;">各指标数据来源设备型号</font>**<font style="background-color:#E8F7CF;">（如 VAPRO6、VDPRO3）；</font>
    - 行为：优先显示在点上方，空间不足时显示在下方；可带箭头指向数据点；多指标时内容可滚动
    - **详细数据来源**：Tooltip 内提供「? 查看详细数据与设备来源」入口，点击后弹框展示该时间点各指标数值及来源设备（体脂率、体重等可能取自不同设备）；**PC 与移动端**均支持上述交互
    - 动画：淡入淡出，约 0.15s
+ **最新点数值**：在折线末端展示最新数据点的数值（含单位）；绘制在图表内部（如末点左侧），避免超出绘图区；当存在双 Y 轴时，与右侧 Y 轴刻度区域保留适当间距，避免与右侧刻度标签重叠
+ **横坐标显示**：不跨年时仅显示「月/日」（如 6/9）；跨年时显示「年/月/日」（如 2025/6/9），便于区分不同年份
+ **时间范围快捷选择**：
    - **管理端/PC**：提供「近一周」「近一月」「近一年」快捷按钮，与开始/结束日期选择器并列；点击后自动填充对应时间范围并刷新图表；选择自定义日期时快捷按钮取消高亮
    - pc端演示demo：[https://dan951781659.github.io/visbody-demo/body-metrics/pc.html#](https://dan951781659.github.io/visbody-demo/body-metrics/pc.html#)
    - **移动端**：仅展示「近一周」「近一月」「近一年」与「**自定义**」按钮；点击「自定义」后从**底部弹框**（Bottom Sheet）形式弹出，内嵌开始日期、结束日期选择器及「确定」「取消」；确定后应用所选时间范围并刷新图表，当前为自定义范围时「自定义」按钮高亮
    - 手机端h5 演示demo：[https://dan951781659.github.io/visbody-demo/body-metrics/mobile.html](https://dan951781659.github.io/visbody-demo/body-metrics/mobile.html)
+ **长区间放大查看**：当时间范围大于约 90 天（如选择「近一年」）时，数据点较密集，需支持放大查看：
    - 支持 X 轴方向**缩放**：滚轮缩放、双指捏合、**拖动框选**区域放大
    - **放大上限**：最大仅展示 **7 天** 数据，小于 7 天后不再继续放大（避免单日过密）
    - 支持 X 轴方向**平移**：放大后可左右滑动/拖拽查看其他时间段
    - **管理端/PC**：页面展示提示「拖动图表可放大查看时间区间」，并提供「**重置缩放**」按钮，点击后所有图表恢复初始视野
    - **移动端**：不展示「拖动图表可放大查看时间区间」文案与「重置缩放」按钮；长区间时仍支持图表 X 轴缩放与平移（双指捏合、拖拽等），重置方式由移动端交互约定（如双指捏合还原或系统手势，可不提供单独重置入口）
+ **同图内某指标当天无数据**：同一图表中，若某一指标在某个日期没有测量数据，其他指标有数据时：
    - **折线**：该指标在该日期不绘制数据点，折线在此处**断开**（不连线），其他指标照常连成折线
    - **Tooltip**：悬停到该日期时，有数据的指标展示数值与单位，**当天无数据的指标展示「—」**
    - **末端数值**：每个指标在折线末端展示的数值，取该指标**最后一个非空数据点**；若最后几天均无数据，则取更早的最后一个有值点
+ **同图内指标量纲差异过大**（如体重 65 kg 与无机盐 3 kg 同属 kg 单位但数值相差悬殊）：
    - 当同一图表内多条指标的数据范围比值超过约 **3 倍** 时，采用**双 Y 轴**展示：数值范围较大的一条或多条用左侧 Y 轴，较小的用右侧 Y 轴，避免小数值曲线被压成平线
    - **右侧 Y 轴样式与左侧一致**：刻度线、网格线、刻度字体大小与颜色与左侧 Y 轴一致，便于阅读
    - 图表标题旁可增加「（左/右轴）」等提示，图例与曲线颜色对应不变，Tooltip 仍展示各指标真实数值与单位
    - （原型/演示）选择体重、体脂肪/蛋白质。即可查看双Y轴的演示效果
+ **其他约定**：
    - 时间范围最长 1 年；超出时前端可提示或截断
    - 仅有一个数据点时仍正常展示该点及 Tooltip
    - 无数据时展示【暂无数据】，未选指标时同样展示【暂无数据】

### 5.4 无数据与异常
+ 查询范围内无数据时，图表区域中央展示 **【暂无数据】**
+ 未选任何指标时，同样展示【暂无数据】

## 六、批量导出数据
### 6.1 功能定位
+ 支持将已选指标在时间范围内的趋势数据批量导出为 **Excel（.xlsx）** 或 **CSV（.csv）** 格式，便于研究、存档或二次分析

### 6.2 功能入口
+ **位置**：数据统计区内，与时间范围选择器同一行，靠右排列
+ **全选本页**：复选框 + 文案「全选本页」，勾选/取消勾选当前报告类型下全部指标卡片
+ **批量导出**：主按钮「批量导出」或「批量导出 ▼」，点击展开下拉菜单
+ **已选条数**：在批量导出按钮旁实时展示「(N)」，如「批量导出 (7)」，未勾选任何指标时不显示数字

### 6.3 导出方式与范围
+ **导出方式**：按勾选导出（仅支持一种模式）
+ **导出范围**：当前已勾选的指标 + 当前时间范围（开始日期、结束日期）
+ **导出格式**：下拉菜单提供「导出为 Excel」「导出为 CSV」两个选项

### 6.4 导出数据格式
+ **表头**：首列为「日期」，其余列为各指标名称（含单位，如「体重(kg)」「体脂率(%)」）
+ **数据行**：每行对应一个日期，列为该日期下各指标的测量值
+ **空值**：未测量或缺失项显示「—」
+ **数值精度**：体成分保留 2 位小数、FMS 取整等

### 6.5 边界条件与交互
+ **<font style="color:#df2a3f;">未勾选任何指标</font>**<font style="color:#df2a3f;">：点击导出时提示「请至少选择 1 个指标后再导出。」</font>
+ **<font style="color:#df2a3f;">未选择时间范围</font>**<font style="color:#df2a3f;">：提示「请选择时间范围后再导出。」</font>
+ **<font style="color:#df2a3f;">导出过程</font>**<font style="color:#df2a3f;">：点击后按钮进入 loading 状态「导出中…」，禁止重复点击</font>
+ **<font style="color:#df2a3f;">导出成功</font>**<font style="color:#df2a3f;">：浏览器下载文件，文件名格式为</font>`**<font style="color:#DF2A3F;">bodily index data</font>**<font style="color:#DF2A3F;">_YYYYMMDD_N.xlsx</font>`<font style="color:#df2a3f;"> 或 </font>`<font style="color:#DF2A3F;">.csv</font>`<font style="color:#df2a3f;">（N 为实际导出行数）</font>
+ **<font style="color:#df2a3f;">导出失败</font>**<font style="color:#df2a3f;">：Toast 提示具体失败原因</font>

### 6.6 CSV 与 Excel 规范
+ **CSV**：UTF-8 编码（含 BOM，便于 Excel 正确识别中文），逗号分隔
+ **Excel**：表头加粗、首行冻结；列顺序与表头一致；数值列、日期列格式正确

## 七、技术规范摘要—参考
### 7.1 接口
+ **指标列表**：`GET /api/users/{userId}/body-metrics`（reportType、customerId、scene 等）
+ **趋势数据**：`GET /api/users/{userId}/body-metrics/statistics`（metricNames、startDate、endDate、customerId、scene 等）
+ 接口语义、跨门店等见原需求文档 **5.1**；；场景 B 当前原型不做门店筛选，接口可预留参数。
+ **批量导出**：可复用趋势数据接口获取数据，前端按选中的 metricNames 及时间范围请求后组装 Excel/CSV 并触发下载；或由后端提供导出接口（可选）。

### 7.2 性能与兼容
+ 列表加载 < 2s，图表数据 < 3s
+ 最多同时展示 10 个指标的趋势
+ 支持 Chrome、Safari、Firefox、Edge；移动端需响应式适配
