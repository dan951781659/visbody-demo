# T5X 接入律动仪 - 产品需求文档

> 本文档基于组员输出的**律动仪设备接入流程图**及**功能清单（Excel）**进行评审与整理，结合评估设备业务逻辑，拆分为 **MVP 版本** 与 **后期优化版本**，作为研发实现与验收依据。  
> **流程图**：见《律动仪接入流程图.md》（Mermaid 源码）或《律动仪接入流程图.html》（浏览器可打开预览）。

---

## 一、需求背景

### 1.1 业务背景

- **主设备定位**：评估设备，核心能力包括：
  - **体成分**：体重、体脂率、肌肉量等
  - **体态**：高低肩、头侧歪、腿型、骨盆等
  - **体适能**：平衡、柔韧性、握力、肺活量等
  - **FMS 评估**：七项动作筛查及总分
- **评估后产出**：根据用户评估数据生成**训练方案**。
- **律动仪接入目标**：评估完成后，输出**使用律动仪完成的训练课程**；用户可在**设备端**查看课程，课程支持**下发到律动仪**进行训练。

### 1.2 目标用户与场景

- **主要用户**：在门店使用评估设备的终端用户（如健身房会员）。
- **使用终端**：评估设备端系统（设备端系统），以及律动仪硬件。
- **核心场景**：完成体成分/体态/体适能/FMS 评估 → 查看推荐课程 → 选择律动仪课程 → 在律动仪上执行训练。

### 1.3 系统角色（与流程图一致）

| 角色 | 职责 |
|------|------|
| **后台管理系统** | 课程创建、配置、上架；同步/下发课程列表；提供课程详情、课程配置、推荐课程接口 |
| **设备端系统** | 评估能力入口、展示推荐课程、请求课程配置/详情、向律动仪下发训练/测试指令、展示训练状态 |
| **律动仪设备** | 接收训练/单动作测试指令，执行并上报执行状态 |

## 二、流程图与功能清单分析结论

### 2.1 流程图正确性分析

组员输出的流程图整体**与当前业务逻辑一致**，具备以下正确点：

| 流程环节 | 结论 | 说明 |
|----------|------|------|
| 课程创建与上架（后台） | ✅ 正确 | 创建课程、组合动作、配置参数、上架、增删改查，是课程可被设备使用的前提 |
| 课程同步与拉取 | ✅ 正确 | 后台同步已上架课程列表，设备端拉取已上架课程，符合「设备展示课程」的来源 |
| 用户开始课程训练 | ✅ 正确 | 设备请求课程配置 → 后台返回动作配置 → 设备下发训练指令 → 律动仪执行并回传状态，形成闭环 |
| 用户查看课程详情 | ✅ 正确 | 请求课程详情、返回动作列表；可选「单动作测试」下发到律动仪，逻辑合理 |
| 评估与推荐课程 | ✅ 正确 | 体成分/体态评估 → 评估结果分析 → 请求推荐课程 → 返回推荐课程列表，与「评估后输出训练方案」一致 |
| 用户点击推荐课程开始训练 | ✅ 正确 | 下发推荐课程训练指令至律动仪，律动仪执行并回传状态 |

**建议补充/澄清：**

1. **评估数据范围**：流程图中仅体现「体成分/体态评估」。当前评估设备还包含**体适能**（平衡、柔韧性、握力、肺活量等）与 **FMS 评估**。推荐课程逻辑应明确：是否将体适能、FMS 结果一并参与推荐算法（建议是），并在需求中写清「推荐依据」的数据范围。
2. **设备端入口**：流程图未细化「用户进入训练课程」的入口。建议在 PRD 中明确：从评估报告/训练方案页的「律动仪课程」入口进入，与「身体指标」「生成训练大纲」等现有入口的关系。
3. **异常与离线**：流程图未体现课程拉取失败、律动仪断连、训练中断等异常分支，建议在 MVP 或后期版本中补充异常与重试策略。

## 三、MVP 版本需求（首期上线）

MVP 目标：完成「评估 → 推荐律动仪课程 → 设备查看 → 下发律动仪训练 → 状态回传」的核心闭环，不依赖后台复杂的课程创作能力，可先采用**预设课程 + 简单推荐规则**。

### 3.1 后台管理系统（MVP）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| M-B-1 | 支持已上架课程的**列表查询**接口，供设备端拉取 | P0 |
| M-B-2 | 支持**单课程详情/动作列表**查询接口，供设备端展示课程详情 | P0 |
| M-B-3 | 支持**课程动作配置**查询接口（顺序/时长/次数等），供设备端下发训练指令 | P0 |
| M-B-4 | 支持**推荐课程**接口：入参含用户评估数据（体成分/体态/体适能/FMS 等，按实际算法需要），返回推荐课程列表 | P0 |
| M-B-5 | 课程数据来源：可先由运营在后台**手工配置并上架**少量律动仪课程，或与现有课程体系打通（若已有） | P0 |

**MVP 可不做**：后台可视化「创建课程、组合多动作视频、配置参数」的完整创作流程，可用配置表/简单后台表单替代。

### 3.2 设备端系统（MVP）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| M-D-1 | **入口**：评估完成后，在训练方案/报告相关页面提供「律动仪课程」或「推荐课程」入口，进入律动仪课程列表 | P0 |
| M-D-2 | **课程列表**：拉取后台「已上架课程」或「推荐课程」列表，展示课程名称、封面、简介等基础信息 | P0 |
| M-D-3 | **课程详情**：支持查看单课程详情与动作列表（请求课程详情接口） | P1 |
| M-D-4 | **下发训练**：用户选择课程并开始训练时，请求该课程配置，向律动仪下发**课程训练指令** | P0 |
| M-D-5 | **训练状态**：接收律动仪上报的**训练执行状态**，在设备端展示（如进行中/完成/失败） | P0 |
| M-D-6 | **推荐课程**：评估完成后可展示「根据您的评估结果推荐」的律动仪课程列表；点击后与普通课程一致，支持下发训练 | P0 |

**MVP 可不做**：单动作测试、课程同步策略（如定时全量/增量）、多课程队列与排队逻辑。

### 3.3 律动仪设备（MVP）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| M-R-1 | 接收设备端下发的**课程训练指令**，按配置执行动作（顺序/时长/次数） | P0 |
| M-R-2 | 向设备端上报**训练执行状态**（如进行中、完成、中断、失败） | P0 |

**MVP 可不做**：单动作测试指令、细粒度进度（如单动作完成百分比）可按实际硬件能力在后期补充。

### 3.4 数据与规则（MVP）

| 项目 | 说明 |
|------|------|
| 推荐依据 | 明确推荐课程接口的入参：至少包含体成分、体态、体适能、FMS 等评估结果（或指标 ID + 数值），便于后续扩展算法 |
| 课程与设备 | 课程与「律动仪」类型绑定，仅展示支持律动仪执行的课程 |
| 失败与重试 | 设备端请求课程列表/配置失败时，有明确提示；下发律动仪失败或断连时，有提示并可重试（具体交互可在原型中细化） |

### 3.5 MVP 验收标准

- [ ] 评估完成后，设备端可进入律动仪课程列表并看到已上架/推荐课程
- [ ] 用户选择课程并开始训练后，设备端能下发训练指令至律动仪，律动仪能按配置执行
- [ ] 律动仪执行状态能回传至设备端并展示
- [ ] 推荐课程接口能根据评估数据（或模拟数据）返回课程列表，并在设备端展示

## 四、后期优化版本需求

在 MVP 闭环稳定后，可迭代以下能力，提升运营效率与用户体验。

### 4.1 后台管理系统（后期）

| 编号 | 需求描述 | 说明 |
|------|----------|------|
| L-B-1 | **课程创作与配置**：创建课程、组合多动作视频、配置顺序/时长/次数等参数 | 与流程图「创建课程」「组合多动作视频」「配置课程参数」一致 |
| L-B-2 | **课程全生命周期**：新增/修改/删除/查询课程，课程上架/下架 | 与流程图「课程上架」及增删改查一致 |
| L-B-3 | **同步策略**：已上架课程列表的同步策略（定时/推送/设备拉取策略优化） | 与流程图「同步已上架课程列表」增强 |
| L-B-4 | **推荐算法增强**：基于体成分、体态、体适能、FMS 的规则或模型优化 | 提升推荐相关性 |

### 4.2 设备端系统（后期）

| 编号 | 需求描述 | 说明 |
|------|----------|------|
| L-D-1 | **单动作测试**：在课程详情内支持「单动作测试」，向律动仪下发单动作测试指令并展示动作测试状态 | 与流程图「单一动作测试」一致 |
| L-D-2 | **课程同步体验**：课程列表的拉取时机、缓存、增量更新等，减少等待与重复请求 | 体验优化 |
| L-D-3 | **多课程与队列**：支持排队、预约下一节课程等（若业务需要） | 按业务再定 |
| L-D-4 | **异常与离线**：律动仪断连、训练中断、超时等提示与重试策略 | 与 2.1 分析建议对应 |
| L-D-5 | **入口与动线**：与「身体指标」「生成训练大纲」等入口统一规划，避免入口分散 | 与 2.1 分析建议对应 |

### 4.3 律动仪设备（后期）

| 编号 | 需求描述 | 说明 |
|------|----------|------|
| L-R-1 | **单动作测试**：接收单动作测试指令，执行并上报动作测试状态 | 与流程图一致 |
| L-R-2 | **进度与细粒度状态**：如单动作完成百分比、当前动作名称等 | 提升体验 |

### 4.4 后期验收标准（概要）

- [ ] 后台可完成课程从创建、配置到上架的全流程
- [ ] 设备端支持课程详情内的单动作测试，律动仪能执行并回传测试状态
- [ ] 课程同步与推荐逻辑可配置或可迭代，异常与断连有明确处理

## 五、与现有需求文档的关系

- **《用户身体指标统计需求文档-最终版》**：身体指标统计、趋势图、生成训练大纲等，与律动仪接入为**并列能力**。律动仪课程是「训练方案」的一种落地形式，建议在「生成训练大纲」或训练方案页增加「律动仪课程」入口，具体入口与文案以产品原型为准。
- **体态/体成分/体适能/FMS**：均为本 PRD 中「推荐课程」的数据输入，推荐接口需能获取到这些评估结果（与身体指标统计的数据来源一致，按客户/用户隔离）。
