# 积分计费机制 - 产品需求文档（PRD）

> **编制说明**：在维塑数字健康平台（WellnessHub）现有设备套餐付费规则基础上，增加积分制度，使 Pro 套餐下的 AI 功能（训练大纲、AI 解读、AI 咨询）按使用次数消耗积分，积分用尽后需用户续费；并与现有 PRO 优惠券规则兼容。  
> **关联页面**：选择套餐页、缴费记录、优惠券管理（新建优惠券）、AI 功能使用入口。

---

## 一、需求背景

### 1.1 业务背景

- 平台当前按**设备数量 × 套餐单价 × 阶梯折扣**对 Basic/Pro 套餐计费，Pro 包含多门店管理及 AI 功能（AI 自由对话、AI 报告解读、AI 个性化训练大纲）。
- 产品预告将在后续版本上线**积分消耗计费机制**：部分 AI 功能按使用次数消耗积分，以更精细地匹配「用多少付多少」的诉求。
- 需在保留现有套餐付费、阶梯折扣、版本升级差价及 PRO 优惠券（免费体验券、代金券）规则的前提下，设计并落地积分体系。

### 1.2 目标用户与场景

| 维度 | 说明 |
|------|------|
| **目标用户** | 已购买/试用 Pro 套餐的门店管理员、教练及 C 端使用 AI 功能的用户 |
| **使用场景** | 使用「训练大纲」「AI 报告解读」「AI 自由对话」时扣减积分；积分不足时引导续费或购买积分包 |
| **核心价值** | 套餐权益与按次消耗相结合，成本可控、计费清晰；与现有优惠券、套餐续费流程一致体验 |

### 1.3 需求目标

1. **积分获取**：明确 Pro 套餐与积分的对应关系，支持按**套餐档位**（如 Prime/Core/Base）或按金额配置；海外多国不同定价时推荐按档位发放，同一档位在不同国家获得相同积分。
2. **积分消耗**：训练大纲、AI 解读、AI 咨询（即 AI 自由对话）每次使用扣减对应积分；扣减规则、单次消耗数值可配置。
3. **积分用尽与续费**：积分余额为 0 或不足单次消耗时，禁止使用上述 AI 功能并引导用户续费套餐或购买积分。
4. **与现有规则兼容**：不改变现有计费公式、阶梯折扣、版本升级差价；免费体验券、代金券的规则保持不变，并在积分上线后与积分发放/消耗逻辑兼容。

---

## 二、现有规则摘要（不变部分）

以下规则在积分功能上线后**继续按现行规则执行**，本文档仅作引用，不做修改。

### 2.1 套餐计费（选择套餐页）

- **计费公式**：总费用 = 设备数量 × 套餐单价 × 阶梯折扣。
- **阶梯折扣**：1–5 台无折扣；6–10 台 9.5 折；11–20 台 9 折；21–30 台 8.5 折；31–50 台 8 折；50 台以上联系销售定制。
- **版本升级差价**：若设备 Basic 版有效期早于升级日期，仅需补缴至 Basic 有效期的差价（无额外费用）。
- **套餐类型**：Basic（设备管理、单门店、测量报告、多账号、3D 体测等）；Pro 在 Basic 基础上增加多门店管理、AI 自由对话、AI 报告解读、AI 个性化训练大纲。

### 2.2 PRO 优惠券规则（新建优惠券）

| 项目 | 说明 |
|------|------|
| **优惠券名称** | 必填，最多 40 字。 |
| **适用类型** | 必选，选「Pro」。 |
| **优惠券类型** | **免费体验券** 或 **代金券**，二选一。 |

**免费体验券**：

- **有效期**：优惠券可被领取的时间段；在此期间用户可领取，未领取则失效。
- **体验时间**：年/月/天三选一填写一个单位；用户使用后，Pro 到期时间 = 当前到期时间 + 体验时间。
- **数量**：可发放总量，为 0 时不可发放。

**代金券**：

- **优惠价格**：支付 Pro 设备费用时直接抵扣的金额。
- **价格单位**：如 USD、CNY 等，与支付货币一致或按产品约定。
- **有效期、数量**：同通用规则（有效期内在支付时可用，数量为 0 不可发放）。

积分规则上线后，上述优惠券的**领取、使用、到期时间计算、代金抵扣**逻辑均不变；**不同优惠券类型与积分的关系**见「3.4 优惠券与积分的衔接」及 3.4.1 汇总表。

## 三、积分计费规则设计

### 3.1 积分与套餐的对应关系

| 项目 | 说明 |
|------|------|
| **积分发放主体** | 以**门店**或**账户**为维度（与现有「按设备数 × 单价」的计费主体一致），具体以产品与研发约定为准；下文均以「该 Pro 账户」代指。 |
| **套餐与积分** | 用户为 Pro 套餐付费时，按**套餐档位（Plan Tier）**或**实付金额**获得对应积分，优先支持按档位配置以兼容海外多国不同定价（见 3.6）。示例（可配置）：<br>• 按档位：Prime → 3000 积分，Core → 2000 积分，Base → 1000 积分（与当地价格无关）；<br>• 或按金额：每 20 美元（或等价货币）→ 2000 积分。 |
| **货币与汇率** | 若采用「按实付金额」发放，需约定积分换算基准（如以 USD 为基准，其他币种按约定汇率折算后再查表）；若采用「按档位」发放，则与币种、汇率无关。 |
| **发放时机** | 支付成功且订单生效后，即时向该账户发放积分；若为续费/升级，同样按当次档位或实付发放。 |

**示例（非最终数值）**：

- 用户为 1 台设备购买 Core 档 20 美元/台/月，按档位配置 Core→2000 积分 → 获得 2000 积分。
- 用户为 10 台设备购买 Prime 档 300 元/台/年，阶梯 9.5 折，实付 2850 元 → 按档位 Prime 配置发放对应积分（如 3000×10 或按产品约定的「多设备发放规则」）。

### 3.6 海外多国不同价格的兼容（按档位发放）

海外同一设备在不同国家定价不同，积分发放采用**按套餐档位发放积分**：与国家、币种、当地价格、付费模式名称无关，**同一档位在同一周期下获得相同积分**。

#### 3.6.1 档位在系统中的定义与识别

| 项目 | 说明 |
|------|------|
| **档位的含义** | 「档位」用于积分发放时唯一确定「送多少积分」，由**适用类型 + 付费周期**两个维度决定，与以下因素**无关**：国家、货币单位、付费模式名称、功能及单价表中的具体价格、是否使用代金券、实付金额。 |
| **如何识别「同一档位」** | 订单支付成功时，系统已知：① 适用类型（如 Pro）；② 用户选择的付费周期（月付 / 年付 / 永久）。用 **适用类型 + 付费周期** 即可唯一确定一个档位，例如：<br>• 档位 **Pro-月付**：任意「Pro + 用户选择月付」的订单；<br>• 档位 **Pro-年付**：任意「Pro + 用户选择年付」的订单；<br>• 档位 **Pro-永久**：任意「Pro + 用户选择永久」的订单。<br>因此，无论该订单来自哪个国家、对应哪个「付费模式详情」、单价是 0.01 还是 20 美元，只要满足「Pro + 月付」，即视为同一档位，发放同一积分。 |
| **与现有「付费模式详情」的关系** | 当前后台「付费模式详情」按国家、货币、付费模式名称配置，且为每个周期配置了「价格/月」「价格/年」「价格/永久」。积分发放**不读取具体价格**，只根据用户下单时选择的**适用类型（Pro）+ 付费周期（月/年/永久）** 查「档位 × 周期 → 赠送积分」配置表并发放。即：所有 Pro 月付订单（不论价格/国家）对应同一积分；所有 Pro 年付、Pro 永久同理。 |
| **若未来存在多档位（如 Prime/Core/Base）** | 若产品将 Pro 细分为多个价格档（如 Prime $30/月、Core $20/月、Base $10/月），可在「付费模式」或定价配置中增加**积分档位**字段（如 Prime / Core / Base），订单支付时取「积分档位 + 付费周期」作为发放依据；同一积分档位+同一周期在不同国家仍发放相同积分。 |

#### 3.6.2 档位 × 周期 → 赠送积分：兑换规则表

采用「档位 × 周期（月/年/永久）→ 赠送积分」配置，**与具体价格无关**。下表为规则示例，数值以实际配置为准。

| 档位（适用类型 + 周期） | 赠送积分（示例，可配置） | 说明 |
|------------------------|--------------------------|------|
| **Pro-月付** | 2,000 | 任意国家、任意单价的 Pro 月付订单，每台设备或每单发放 2000 积分（按产品约定的发放单位：每设备或每账户） |
| **Pro-年付** | 20,000 | 任意国家、任意单价的 Pro 年付订单，发放 20000 积分；可按「约 10 个月等效」或运营策略设定 |
| **Pro-永久** | 50,000 | 任意国家、任意单价的 Pro 永久订单，发放 50000 积分；可按「约 25 个月等效」或运营策略设定 |

- **多设备**：若用户一次为 N 台设备购买同一档位（如 Pro-月付），发放积分 = 单档位积分 × N，或按产品约定的阶梯/上限规则（如设上限）执行。
- **配置落地**：后台需提供「积分档位配置」页或表，维护上述映射；若仅有 Pro 且无 Prime/Core/Base，则档位枚举为 Pro-月付、Pro-年付、Pro-永久 三条即可。

#### 3.6.3 代金券（价格优惠券）与积分的对应规则

在**按档位发放积分**的前提下，代金券只影响实付金额，**不影响发放的积分数**。

| 规则项 | 说明 |
|--------|------|
| **原则** | 用户支付时选择的是「Pro + 某周期」（即某一档位），系统按**档位**发放**固定积分**，与是否使用代金券、实付金额多少无关。 |
| **举例** | 用户为 1 台设备购买 Pro 月付，单价 20 美元；若不用券，实付 20 美元，获得 2000 积分；若使用 5 美元代金券，实付 15 美元，**仍获得 2000 积分**（因档位仍为 Pro-月付）。 |
| **原因** | 若按实付金额折算积分，代金券会导致「付得少、拿的积分也少」，体验差且易引发客诉；按档位发放则逻辑简单、口径统一，且与「同一档位在不同国家获得相同积分」一致。 |
| **实现要点** | 支付成功回调或订单入账时，用「适用类型 + 付费周期」查档位→积分表发放，**不参与实付金额计算**；代金券仅在前端/支付流程中抵扣金额，不传入积分发放逻辑。 |

#### 3.6.4 小结：三个问题的直接回答

| 问题 | 答案 |
|------|------|
| **系统中如何知道是同一个档位？** | 用**适用类型（Pro）+ 用户选择的付费周期（月/年/永久）** 唯一确定档位，不依赖国家、付费模式名称、价格。订单支付时必有这两项信息，据此查「档位 → 赠送积分」即可。 |
| **是否所有 Pro 月付/年付/永久，不管设置多少价格，对应积分数都一样？** | **是**。同一档位（如 Pro-月付）在所有国家、所有付费模式、任意单价下，赠送的积分数一致，由配置表唯一决定。 |
| **代金券（价格优惠券）如何对应积分？** | **按档位发放时，代金券不影响积分**。使用代金券后仍按「Pro + 所选周期」发放该档位对应的固定积分，不按实付金额折算。 |

### 3.2 积分消耗功能与单次消耗量

| 功能（产品名称） | 对应能力说明 | 单次消耗积分（示例，可配置） |
|------------------|--------------|------------------------------|
| **AI 个性化训练大纲** | 根据用户体测报告与关键问题生成个性化训练方案及里程碑计划 | 如 50 积分/次 |
| **AI 报告解读** | 根据体测报告综合分析身体关键异常、优先改善建议 | 如 30 积分/次 |
| **AI 自由对话（AI 咨询）** | 用户对体测报告指标进行多轮对话式解读 | 如 10 积分/次（或按对话轮次/按次封顶，以配置为准） |

- **扣减时机**：用户点击「生成训练大纲」「生成/查看 AI 解读」「发起/继续 AI 咨询」并**成功调用服务**时扣减；若调用失败（含网络、服务异常）不扣减。
- **扣减顺序**：同一账户下积分全局共用，按使用时间顺序依次扣减，扣减后实时更新余额。
- **扩展约定**：新增消耗积分的 AI 功能（如 AI 训练计划生成）时，在「单次消耗配置」中增加功能标识与单次消耗量即可，入口处复用现有扣减与不足拦截逻辑；不写死功能列表，详见 9.1。

### 3.3 积分余额与使用限制

| 规则项 | 说明 |
|--------|------|
| **余额展示** | 在管理后台（及 C 端若适用）展示当前账户「可用积分」；在 AI 功能入口旁可展示本次操作将消耗的积分数（如「本次将消耗 50 积分」）。 |
| **不足时拦截** | 当 可用积分 < 本次操作所需积分 时，不允许发起该次 AI 功能调用；前端置灰或点击后弹窗提示「积分不足，请续费或按量购买积分」，并提供「续费 Pro」与「按量购买积分」两个入口。 |
| **为 0 时** | 同上，视为不足；提示文案可统一为「积分已用尽，请续费套餐或按量购买积分」。 |
| **续费与按量购买** | 用户可选择：① **续费 Pro**（选择套餐）：按档位获得赠送积分；② **按量购买积分**：在 Pro 有效期内单独购买积分，支付后即时到账，无需续费。详见 3.7。 |

### 3.7 积分用尽后的按量购买

积分用尽或不足单次消耗时，用户除续费 Pro 外，可选择**按量购买积分**，按需获得积分后继续使用耗积分 AI 功能。

#### 3.7.1 适用条件与前置条件

| 项目 | 说明 |
|------|------|
| **触发场景** | 可用积分 < 本次操作所需积分（含积分为 0）；用户点击耗积分 AI 功能时被拦截，或主动进入「积分中心」购买。 |
| **前置条件** | **当前账户 Pro 仍在有效期内**。若 Pro 已到期，须先续费 Pro 后方可购买或使用积分；按量购买不延长 Pro 到期时间，仅增加积分余额。 |
| **与续费的关系** | 按量购买是**补充手段**：用户不想立刻续费套餐时，可先买积分应急；后续仍可通过「选择套餐」续费，续费时按档位正常获得赠送积分。 |

#### 3.7.2 按量购买的产品形态与定价

| 项目 | 说明 |
|------|------|
| **产品形态** | 提供若干**积分档位**供用户选择（如 500 积分、1000 积分、2000 积分、5000 积分），或支持**自定义数量**（在限定范围内，如 100～10000 步长 100）；每档/每单位对应一个价格，支付后该笔积分即时到账。 |
| **定价方式** | 按**国家/币种**配置「积分数量 → 价格」或「单价」；可与套餐定价分离，例如：100 积分 = 1 美元（或当地等价）。支持多档位时可为不同档位设置阶梯单价（如买得越多单价略降）。 |
| **配置要求** | 后台需支持：按量购买档位列表（数量 + 对应价格）、可选按国家/币种配置、单次购买数量上下限（若有）。 |

#### 3.7.3 发放、消耗与有效期

| 项目 | 说明 |
|------|------|
| **发放** | 支付成功后，**即时**向当前账户发放对应数量积分；发放类型记为「按量购买」（与「套餐赠送」区分，便于流水与报表）。 |
| **消耗** | 与套餐赠送的积分**共用同一积分池**，消耗规则一致：训练大纲、AI 解读、AI 咨询等按配置单次扣减，先到账先扣或统一池子无区分。 |
| **有效期** | **与 3.5 一致**：按量购买获得的积分，有效期仍与**该账户 Pro 到期时间**一致；Pro 到期则积分清零，不因「按量购买」而延长 Pro 或单独延长积分有效期。 |

#### 3.7.4 入口与流程

| 项目 | 说明 |
|------|------|
| **积分不足时** | 弹窗或提示页提供两个入口：① **续费 Pro**（跳转选择套餐）；② **按量购买积分**（跳转积分购买页或弹窗内选择档位并支付）。 |
| **常驻入口** | 在「选择套餐」页、账户/积分中心、或 AI 功能入口旁提供「按量购买积分」入口，便于用户提前囤积或用尽后主动购买。 |
| **流程** | 选择积分档位（或输入数量）→ 确认价格与到账说明 → 支付 → 支付成功 → 积分到账，可立即使用耗积分 AI。 |

#### 3.7.5 规则小结

- **谁可买**：当前 Pro 在有效期内的账户。
- **买多少**：按后台配置的档位或自定义范围选择，支付对应金额。
- **何时到账**：支付成功即时到账，与套餐赠送积分合并显示。
- **何时失效**：随 Pro 到期一并清零，不单独延期。
- **与续费**：按量购买不替代续费；续费 Pro 仍按档位获得赠送积分，两者可并存。

### 3.4 优惠券与积分的衔接

#### 3.4.1 不同优惠券类型与积分的关系汇总

| 优惠券类型 | 对 Pro 权益的影响 | 是否赠送积分 | 体验期/支付期内是否可消耗积分 | 与积分发放的对应规则 |
|------------|-------------------|--------------|-------------------------------|------------------------|
| **免费体验券** | 延长 Pro 到期时间（当前到期时间 + 体验时间）；不涉及支付。 | **不赠送积分** | **体验期内不开放**消耗积分的 AI 功能（训练大纲、AI 解读、AI 咨询），仅开放不耗积分的 Pro 能力（多门店管理、3D 体测报告等）。 | 不触发积分发放（无支付行为）。体验期结束后用户若续费 Pro，按正常档位规则获得积分。 |
| **代金券** | 用户支付 Pro 时抵扣金额，实付减少；不改变所选套餐档位与周期。 | **按档位发放固定积分**，与是否使用代金券、实付多少无关。 | 支付成功后积分到账，可正常消耗（训练大纲、AI 解读、AI 咨询）。 | 支付成功时按「Pro + 付费周期」查档位→积分表发放，**代金券不减积分**。详见 3.6.3。 |

**简要结论**：

- **免费体验券**：只延长时间、不送积分；体验期内不能用耗积分的 AI 功能，续费后按档位正常获得积分。
- **代金券**：只减价格、不减积分；按档位发放固定积分，与实付金额无关。

#### 3.4.2 规则说明（与上表一致）

- **免费体验券（方案 A）**：用户使用免费体验券后，仅延长 **Pro 到期时间**（现有逻辑不变）；**不自动赠送积分**。**体验期内不开放消耗积分的 AI 功能**（训练大纲、AI 解读、AI 咨询），仅开放不耗积分的 Pro 能力（如多门店管理、3D 体测报告等）。体验期结束后，若用户续费 Pro 并按档位获得积分，方可使用上述 AI 功能。
- **代金券**：采用「按档位发放积分」时，用户使用代金券抵扣 Pro 支付金额，**仅减少实付金额，不减少发放的积分**。系统仍按**档位（Pro + 付费周期）**发放该档位对应的固定积分，详见 3.6.3。

### 3.5 积分有效期与清零

| 项目 | 说明 |
|------|------|
| **有效期定义** | 积分有效期与**该设备（或该账户下设备）的 Pro 到期时间**一致，不单独设置积分日历有效期。 |
| **清零规则** | **设备 Pro 到期则对应积分清零**。即：以设备维度时，某设备 Pro 到期则与该设备关联的积分余额清零；以账户/门店维度时，该账户下所有设备 Pro 均到期则积分清零。 |
| **提醒** | 建议在 Pro 即将到期前（如 7 天、1 天）在后台或通知中提醒「积分将随 Pro 到期清零，请及时使用或续费」。 |

## 五、非功能需求与边界

### 5.1 数据与一致性

- 积分发放、扣减须**事务一致**：支付成功则积分必到账；扣减成功则流水可追溯，避免超扣或漏扣。
- 高并发下积分扣减需防超卖（如乐观锁或预扣+确认机制），具体由技术方案保证。

### 5.2 规则说明与用户沟通

- 在「选择套餐」页、帮助中心或弹窗中提供**积分计费规则说明**（获取方式、消耗方式、用尽与续费、与优惠券的关系），与图 1 中现有「积分计费机制升级预告」表述一致并细化。
- 积分规则正式上线后，**新老账户**均适用新规则；上线前已购买的套餐权益在规则说明中已承诺「按现行规则执行」的，在约定过渡期内可保留旧则，具体以正式公告为准。

### 5.3 异常与客诉

- **扣减失败/重复扣减**：若因系统异常导致重复扣减，需支持按流水核查并补回积分；若未扣减但服务已提供，按运营策略决定是否补扣或豁免。
- **退款**：用户申请退款时，已发放的积分按策略处理（如按比例收回、或仅收回未使用部分），需在退款策略中明确并落地。

## 七、附录：积分规则说明（用户向文案参考）

以下可作为「积分计费规则说明」的对外表述参考，具体数值以正式上线说明为准。

- **积分获取**：购买或续费 Pro 套餐时，根据套餐档位（如 Prime/Core/Base）或实际支付金额获得相应积分，具体以支付页说明为准；海外不同国家同一档位获得的积分一致。
- **积分消耗**：使用「AI 个性化训练大纲」「AI 报告解读」「AI 自由对话」时将按次消耗积分，单次消耗量以功能页展示为准。
- **积分用尽**：积分用尽后，上述 AI 功能将无法使用。您可选择**续费 Pro 套餐**（按档位获得赠送积分）或**按量购买积分**（在 Pro 有效期内单独购买积分，即时到账），继续使用。
- **按量购买**：在 Pro 有效期内可随时「按量购买积分」，选择积分档位并支付后即时到账，与套餐赠送积分合并使用；积分仍随 Pro 到期清零。详见文档 3.7。
- **与优惠券**：**免费体验券**不赠送积分，体验期内不可使用消耗积分的 AI 功能；**代金券**仅减实付金额，仍按档位获得固定积分，不减积分。详见文档 3.4.1。
- **积分有效期**：积分随设备 Pro 到期而清零；Pro 续费后新获得的积分在新周期内有效。
- **适用范围**：积分适用范围、扣减方式、功能覆盖及具体数值，以正式上线说明为准。

## 八、方案评审与优化建议

### 8.1 当前方案的合理性评价

| 维度 | 评价 | 说明 |
|------|------|------|
| **按档位发放** | 合理 | 与海外多国不同定价解耦，同一档位全球积分一致，运营与用户理解简单；代金券不减积分，避免「用券反而少拿积分」的客诉。 |
| **积分随 Pro 到期清零** | 合理 | 与订阅生命周期一致，无永久积分遗留，续费动机清晰。 |
| **免费体验券不送积分、体验期不开放耗积分 AI** | 合理 | 边界清晰，防薅羊毛；体验期后可引导正式续费获得积分。 |
| **档位 = 适用类型 + 周期** | 合理 | 订单支付时必有这两项，无需读价格即可发积分，实现简单。 |

整体上当前方案**逻辑自洽、易落地、易扩展**，可直接作为一期实现依据。

### 8.2 建议优化与提前约定

| 优化项 | 建议 | 目的 |
|--------|------|------|
| **混合订单（多档位同单）** | 若用户一次勾选多台设备且**部分选月付、部分选年付**，建议约定：按**每个设备 + 其选择的周期**分别确定档位并发放积分（即订单行级别发放），而非整单只认一个档位。在订单/支付设计中预留「行项目档位」即可。 | 避免「混单只发一种积分」导致少发或多发争议。 |
| **按量购买与积分包** | 积分用尽后的按量购买逻辑已在 3.7 定义（前置条件 Pro 有效、即时到账、有效期随 Pro 到期）。若后续增加「积分包」预售档（如大额包、季包），建议与按量购买共用同一发放类型或区分子类型，统一遵守「随 Pro 到期清零」；单次购买上限、转赠等以届时产品说明为准。 | 与 3.7 一致，避免两套有效期或发放口径。 |
| **多档位（Prime/Core/Base）落地** | 若产品将 Pro 拆为多价格档，**付费模式详情**或定价配置中需增加**积分档位**字段（如 Prime/Core/Base），订单支付时传「积分档位 + 周期」而非仅「Pro + 周期」。当前文档 3.6.1 已说明，建议在需求评审时与后端约定字段名与枚举。 | 保证扩展多档位时无需大改发放逻辑。 |
| **体验期内积分展示** | 免费体验券用户体验期内不开放耗积分 AI，建议明确：体验期内**是否展示积分余额**。若展示，需提示「体验期内不可使用耗积分 AI，续费后开放」；若不展示，可避免误解。二选一在交互规范中定即可。 | 避免用户误以为「有积分但不能用」是 Bug。 |

## 九、扩展场景与兼容性

以下为后续可能出现的扩展场景，说明**当前付费与积分设计如何兼容**，以及需要做的最小补充。

### 9.1 新增消耗积分的 AI 功能（如 AI 训练计划生成）

| 项目 | 说明 |
|------|------|
| **场景** | 产品后续上线新的 AI 能力，例如「AI 训练计划生成」「AI 康复建议」等，同样按次消耗积分。 |
| **兼容方式** | **无需改发放逻辑**。在「单次消耗配置」中增加**可配置项**：功能标识（如 `ai_training_plan`）、功能名称、单次消耗积分。新增 AI 功能时在后台增加一条配置，前端在该功能入口复用现有逻辑：查余额、扣减、不足时拦截并引导续费。 |
| **建议约定** | ① 所有「消耗积分的 AI 功能」统一走同一套扣减与拦截逻辑；② 功能列表与单次消耗量**可配置、可扩展**，不写死在前端；③ 在 PRD/配置表中预留「功能类型」或「能力 ID」，便于后续按类型做统计或差异化策略。 |

**结论**：当前设计**天然兼容**新增 AI 功能，只需扩展配置表与入口鉴权。

### 9.2 用户购买多种类型设备（多产品线）

| 项目 | 说明 |
|------|------|
| **场景** | 同一用户/门店拥有多类设备（如体态设备 T50、律动仪、其他硬件），每类设备可能有不同的 Pro 价格或功能包，用户为不同类型设备分别购买/续费 Pro。 |
| **兼容方式（推荐）** | **档位扩展为「产品线（可选）+ 适用类型 + 周期」**。若当前仅一种 Pro（通用），则无需改；若未来按产品线区分（如「Pro-体态」「Pro-律动仪」），在付费模式或订单中增加「产品线/设备类型」维度，积分档位配置表增加对应条目（如 Pro-体态-月付→2000，Pro-律动仪-月付→1500）。**积分池建议仍为账户/门店统一**：不同设备购买的积分进入同一池子，所有耗积分 AI 从同一余额扣减，这样用户理解简单、实现也简单。 |
| **备选** | 若希望「体态设备积分」与「律动仪积分」分池使用，需在 3.1 中增加「积分类型」或「积分池」概念，发放与消耗时带类型，逻辑和配置复杂度会明显上升，建议一期不做，必要时再扩展。 |

**结论**：多设备类型可通过**扩展档位维度（产品线）**兼容；积分池统一保持简单。

### 9.3 多档位定价（Prime / Core / Base）正式上线

| 项目 | 说明 |
|------|------|
| **场景** | 海外或国内将 Pro 细分为 Prime（$30/月）、Core（$20/月）、Base（$10/月）等，同一国家存在多种 Pro 价格档，用户按档位选择购买。 |
| **兼容方式** | 文档 3.6.1 已说明：在「付费模式详情」或定价配置中增加**积分档位**字段（Prime/Core/Base），订单支付时传入「积分档位 + 周期」。兑换规则表扩展为例如：Prime-月付→3000，Core-月付→2000，Base-月付→1000，年付/永久同理。发放逻辑不变：仍按「档位 + 周期」查表发放，与价格、国家无关。 |
| **结论** | 当前按「适用类型 + 周期」的档位设计，**只需把「适用类型」细化为「积分档位」枚举**即可兼容，无需改核心流程。 |

### 9.4 按量购买与积分包、积分转赠

| 项目 | 说明 |
|------|------|
| **按量购买** | 积分用尽后的按量购买逻辑已在 **3.7** 完整定义：Pro 有效期内可单独购买积分档位，支付后即时到账，与套餐赠送积分共用池子，有效期随 Pro 到期清零。 |
| **积分包（大额/预售档）** | 若后续增加「积分包」形态（如 5000/10000 积分包、季包等），可与按量购买共用发放与消耗逻辑，仅定价与档位配置不同；仍遵守 3.7.3 的有效期规则。 |
| **转赠** | 若支持积分在账户间转赠，需增加转出/转入流水类型、扣减与发放校验、风控规则。建议作为二期能力，一期不承诺；以届时产品说明为准。 |
| **结论** | 按量购买已纳入 3.7；积分包为按量购买的扩展档位形态；转赠建议后期单独 PRD。 |

### 9.5 扩展兼容性小结

| 扩展场景 | 是否兼容 | 需做的最小改动 |
|----------|----------|----------------|
| 新增 AI 功能（如 AI 训练计划生成） | 是 | 消耗配置表与入口增加可配置项，复用扣减与拦截逻辑 |
| 用户购买多种类型设备 | 是 | 档位增加「产品线/设备类型」维度（可选）；积分池建议仍统一 |
| 多档位 Prime/Core/Base | 是 | 付费模式增加「积分档位」字段，兑换表扩展档位枚举 |
| 积分用尽后按量购买 | 是（已纳入 3.7） | 见 3.7 与 F5b、F9；积分包可作按量购买扩展档位 |
| 积分转赠 | 后期 | 单独方案与风控，二期考虑 |

当前付费与积分模式在**不改变「按档位发放、按次消耗、随 Pro 到期清零」核心**的前提下，通过**配置化消耗列表**和**档位维度扩展**即可覆盖上述扩展场景。
