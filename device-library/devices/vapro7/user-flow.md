# VAPro7 使用流程

## 最后更新依据

- 基于 `Visbody-A Pro7` 产品使用说明书中的安装、使用准备、使用须知、报告和设置章节整理

## 确定事实

### 1. 安装准备

- `已确认`：设备需要完成拆箱、地垫铺设、转台放置、支架安装、镜体安装、转台连接和通电
- `已确认`：设备支持连接打印机
- `已确认`：设备可接入有线网络，也支持 `Wi-Fi`
- `已确认`：正式使用前需要联系维塑售后注册管理端管理员账号

### 2. 使用前准备

- `已确认`：管理端账号注册需提供设备 `SN` 号、邮箱、公司名和店名
- `已确认`：管理端可上传纸质报告 logo
- `已确认`：管理端可配置报告列表用户 ID 和纸质报告用户 ID 的加密方式

### 3. 开机与初始化

- `已确认`：通过设备右侧电源键开机
- `已确认`：启动时间约 `30` 秒
- `已确认`：开机后设备执行自检和自动校准
- `已确认`：自动校准结束后进入待机界面；唤醒后进入测量项目选择

### 4. 转台身高与体重（设备能力）

- `已确认`：用户站转台期间可同步完成身高（Hub 推导，见综合 PRD §7.4）与体重采集
- `已确认`：采集中以 Toast 提示完成，不中断后续体成分/体态/体围流程
- `已确认`：若 Hub 配置「身高是否需要确认」，整轮测量结束后进入身高二次确认页，再进测量已完成页
- `已确认`：独立体重入口在项目选择页（Hub `weightStandaloneEnabled`）；测完 **20 秒无操作**或手动返回 → 项目选择页；**不参与**测量循环与「继续下一项」推荐（综合 PRD §6.2）

### 5. 测量前准备

- `已确认`：用户测量前应脱去外套，并尽量穿着轻薄贴身服饰

### 6. 体态评估流程

- `已确认`：用户走上转台并静止 `2` 秒启动测试
- `已确认`：用户需双手自然下垂站立
- `已确认`：用户需保持头部不动
- `已确认`：设备在倒计时 `321` 后随转台旋转一周完成体态评估

### 7. 身体成分测量流程

- `已确认`：用户走上转台并静止 `2` 秒启动测试
- `已确认`：用户先双手自然下垂站立，进行体重测量
- `已确认`：随后双手握紧扶手，双臂张开 `45°`
- `已确认`：保持头部不动，在倒计时 `321` 后随转台旋转一周完成测量

### 8. 查看报告

- `已确认`：用户可扫描设备端二维码，在微信中打开“脂肪管理局”公众号查看报告
- `已确认`：新用户需先注册账号，再进入移动端报告页面

### 9. 打印报告

- `已确认`：方法一为通过网页版管理端打印
- `已确认`：通过管理端打印时，电脑需通过 `USB` 或云端与打印机连接
- `已确认`：方法二为通过设备端打印
- `已确认`：设备端打印需要将打印机连接到设备后方 `USB` 接口
- `已确认`：用户在报告列表中选中报告后，可按电源键执行打印
- `已确认`：设备支持手动打印和自动打印模式

### 10. 设置与维护

- `已确认`：待机页 status-bar 或长按 `-` 键进入设置页（项目选择页无设置入口）
- `已确认`：设置结束后，连续快速按 `2` 次电源键回到首页
- `已确认`：设置模块包含声音、亮度、交互模式、打印、按键指引、设备检测、更新、脚部加热等功能；**身高/体重/项目配置**由 WellnessHub 下发，不在设备设置页编辑

### 11. 测量完成与下一项（产品目标，见综合 PRD §6）

- `已确认`：各测量项目完成后进入统一「测量已完成」页，含举手引导
- `已确认`：非末项可「继续下一项」；**本轮最后一项**仅「完成测量」（进入扫码报告链）
- `已确认`：「继续下一项」按 Hub 首页顺序**环形**遍历已启用项目；起点为用户从项目选择页点击的第一项
- `已确认`：独立体重**永不**作为「继续下一项」推荐目标

### 12. 设备端与外部系统协同链路

- `已确认`：管理端账号注册是设备投入使用前的前置条件
- `已确认`：设备端二维码用于引导用户进入移动端查看报告
- `已确认`：网页版管理端支持报告打印
- `已确认`：打印链路依赖设备兼容的打印机型号

## 设计含义

- `推断`：`VAPro7` 的关键体验不是单页功能，而是一条端到端链路，包含设备准备、现场测量、结果查看和结果输出。
- `推断`：设备的开机自检、自动校准和测量姿势约束说明硬件状态对体验影响较大，任何新增功能都应避免破坏测量主流程的稳定性。
- `推断`：二维码查看报告代表设备端会把一部分结果消费场景导向手机端，因此设备端新增功能需要考虑跨端接续，而不是强行把所有信息留在设备上完成。
- `推断`：账号注册、logo 配置、ID 加密都发生在管理端，说明某些设备能力实际依赖后台配置，后续需求评估时应同步梳理后台改动。
