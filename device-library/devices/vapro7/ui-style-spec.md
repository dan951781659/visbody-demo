# VAPro7 设备端 UI 规范

## 来源

- `/Users/kean/Downloads/LanhuProject/`
- 现有设备端 demo：`/Users/kean/Documents/体测线/vapro7-demo/`

## 使用原则

- 蓝湖导出只作为 **UI 规范来源**，不作为业务流程规范来源。
- 设备端新增业务流程、跳转关系、配置开关、`data-*` 驱动逻辑不得因视觉改造被删除、合并或回退。
- 后续设备端页面如未单独指定，以本文件作为默认视觉规范。

## 画布与适配

- 设计基准画布：`1080 × 1920`
- 页面默认形态：纯黑单屏、强引导、大卡片、低层级信息组织
- 当前 demo 实现应继续保留现有状态驱动逻辑，视觉层按 1080 规范折算到设备页壳层

## 色彩体系

- 页面背景：纯黑 `#000000`
- 主操作卡：蓝色 `#1D66E6`
- 强调底条/深色蓝：`#104EBB`
- 次级卡：灰色 `#555555`
- 次级深底：深灰 `#2C2C2C`
- 主文字：白色 / 近白 `#FFFFFF`、`#E7E9F0`
- 辅助文字：`#CCCCCC`
- 弱提示文字：`#999999`
- 高亮提示色：橙色 `#FF791F`
- 分割线：`#565656`

## 排版层级

- 顶部返回文案：蓝色，清晰独立，不与标题并排混排
- 页面主标题：大号白字，单屏主视觉下优先居中或强左对齐
- 副标题：浅灰说明文案，层级低于标题，不超过两行核心说明
- 卡片标题：高对比白字
- 卡片副文案：浅灰正文
- 手势/动作提示：比副文案更醒目，但低于主标题

## 卡片与间距

- 大卡圆角基准：`32px`
- 设置、项目选择、完成态等主卡片统一使用大圆角实体块，不使用渐变 demo 卡
- 首页/项目选择页：
  - 第一张主卡为蓝色
  - 后续卡片为灰色
  - 左侧可带竖向强调条或图标区
- 页面左右边距、标题区、卡间距保持稳定，不随单页随意变化

## 图标规范

- 优先使用白色设备风图标或已有设备素材
- 禁止使用 emoji 作为正式设备端图标
- 新增图标应保持统一线条粗细、统一留白和统一视觉重心

## 页面模板

### 1. 项目选择页

- 顶部：状态栏、返回入口、大标题
- 中部：纵向大卡列表
- 底部：返回首页倒计时提示

### 2. 测量准备页

- 顶部：状态栏、返回、大标题、简短说明
- 中部：准备清单区
- 下部：手势确认区
- 底部：动作卡（退出 / 下一步）

### 3. 引导页 / 识别页

- 顶部：状态栏、返回、大标题、副说明
- 中部：主视觉识别舞台
- 底部：单主操作区或状态提示区

### 4. 倒计时页

- 主体：姿势示意 + 大数字倒计时
- 次级：注意事项 / 开始提示

### 5. 测量中页

- 主视觉区为唯一焦点
- 状态 chip、进度条、提示文案围绕主视觉组织

### 6. 完成页

- 顶部：完成标题
- 中部：完成摘要
- 底部：后续动作卡

## 禁止项

- 不要使用外层 demo 感渐变舞台作为正式设备页面背景
- 不要使用紫色主品牌风格
- 不要使用 emoji 图标
- 不要把蓝湖静态占位文案或条目数量直接硬编码进当前业务页面
- 不要为了贴视觉稿而修改已有业务流程和逻辑分支
